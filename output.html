Backup Steps

Step 1: On target Windows server, create SMB share (if not already available) at \\windows-target\backups

Step 2: On source Windows server, map network drive

net use Z: \\windows-target\backups /user:backupuser <password>

Step 3: Create timestamped folder

$timestamp = Get-Date -Format "yyyyMMdd"
$targetFolder = "Z:\windows-source\$timestamp"
New-Item -ItemType Directory -Path $targetFolder

Step 4: Back up source folders to defined backup location

Step 4.1: Run Robocopy from D:\Projects to target folder

robocopy D:\Projects $targetFolder /MIR /LOG:C:\backup\backup.log

Step 4.2: Run Robocopy from C:\Backup\Configs to target folder

robocopy C:\Backup\Configs $targetFolder /MIR /LOG:C:\backup\backup.log

Step 5: Verify the files on target Windows server

Step 6: Unmap network drive

net use Z: /delete

Step 7: Implement retention policy on Windows target

Example PowerShell one-liner:

Get-ChildItem -Path "Z:\windows-source" -Directory | Where-Object { $_.LastWriteTime -lt (Get-Date).AddDays(-10) } | Remove-Item -Recurse -Force

Step 8: Automate this backup with Windows Task Scheduler

The scheduled task should run a PowerShell script or batch file that:

  • Map network drive
  • Create new timestamped folder
  • Run Robocopy
  • Run retention cleanup
  • Unmap network drive
